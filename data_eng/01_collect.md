# 박스 01 — 수집 (소스 + Ingestion)

## 목적
RAG에 넣을 국가유산 원천 데이터를 외부 소스에서 가져와 누적한다.

## 현재 상태
🔄 진행 중 — 국가유산검색 목록+상세 API 2단계 호출로 ~15,000개 확장 수집 중 (`fetcher_search.py` 실행)
- 기수집 627개 (4대궁·종묘 128 + 국보 499) 기반 위에 시도 종목 포함 전 종목으로 확장
- 보물 수집은 IP 차단 경험 후 호출 간격 1초로 재개

## 산출물
- 데이터: `practice/data/heritages.json` (627개)
  - 4대궁·종묘: 128개 (광화문, 근정문 등 부속 위주)
  - 국보: 499개 (숭례문, 원각사지 등 메인)
  - 백업 5개: 경복궁/창덕궁/종묘/숭례문/불국사 (location 있음)
- 코드:
  - `practice/rag/fetcher.py` — 4대궁+종묘 (Stage 2a/2b)
  - `practice/rag/fetcher_search.py` — 국가유산검색 API (Round 1)
  - `practice/rag/merge_backup.py` — backup → heritages.json 병합

## 입력 → 출력

```
국가유산청 OPEN API 7종 (XML)
4대궁·종묘 상세 API
  ↓ fetcher.py / fetcher_search.py
heritages.json (raw 가까운 상태)
```

| 종목 우선순위 | 국가 종목 (국보·보물·사적·명승·천연·무형·민속·등록) | 본선까지 ~5K 목표 |

## 보조 소스 활용 계획 (계획만 — 구현 X)

> 627개(→15K) 주 데이터가 못 메우는 갭을 보강할 후보. 실제 fetcher는 박스 03 안정화 후.

| 후보 소스 | 메우는 갭 | 커버 질문 유형 | 통합 방식 | 키 | 우선순위 |
|---|---|---|---|---|---|
| 한국역대인물 (G) | 인물 정보 갭 | "정조는 누구?", "이성계가 세운…" | XML 파일 1회 다운 → 인덱싱 | 불필요(파일) | 🟠 2순위 |
| **한국문화재용어사전** | 용어·도메인 지식 갭 | "법궁이 뭐야?", "보물·국보 차이", "다포식?" | CSV 1회 다운 → **별도 컬렉션** | 불필요(파일) | 🟠 2순위 |
| e뮤지엄 (F) | 유물 단위 디테일 | "심화 설명", 도감 | OpenAPI → 유물 단위 | 필요 | 🟡 3순위 |
| 특일정보 (H) | 시의성 | "오늘의 유산"(한글날→훈민정음) | OpenAPI → 날짜 매핑 | 필요 | 🟡 3순위 |

### 용어사전 상세 (2026-05-31 조사 완료)
- 정식명: `한국국제교류재단_한국문화재용어사전_국영문_20240215`
- 제공: 한국국제교류재단(KF) / 형식: CSV(+OpenAPI 옵션)
- 필드: 시작자음·**한자**·독음·**설명**(국영문) / 규모: ~2,482개 (건축 395·고고학 884·미술사 1,203)
- 라이선스: **이용허락범위 제한 없음, 무료** / 키: CSV 다운로드는 불필요
- 출처: https://www.data.go.kr/data/3073111/fileData.do
- 비고: 유산 데이터와 성격이 달라 **별도 컬렉션** 권장. 한자 필드 보유 → 정제 ④(한자 처리) 결정과 연동.
