# 🏆 PART A. 보완 포인트 — "팀 한계를 이렇게 메운다" (핵심·표지)

> 빌드는 팀이 했다. **검증·품질·진단**이 우리 자리. 아래는 전부 팀 문서 15절 한계와 1:1 대응.

## 1. 검색 ranking 미검증 → 측정 가능하게 만듦 (팀 한계 ②)

- 🔴 **문제**: 벡터 검색이 *제대로 찾는지* 검증 안 됨 (팀 자백: "ranking 충분히 검증 안 됨")
- 🔍 **원인 진단**: 평가 **정답 라벨 자체가 없어** 측정 불가 상태였음 — 지표가 깨져 있어 *재기만 하면 헛수치*
- ✅ **보완**: `expected_heritage` 정답셋 라벨링 → 🔬 단일 질문 **Recall@3 = 100%(23/23)**, 비교 질문 두 유산 적중 **83%(15/18)** 실측 ⚠️(5,743 캐시 기준)
- 📤 **산출물**: 팀에 **ranking 검증 리포트**로 전달 → "검증 안 됨"(한계②)이 "수치로 검증됨"이 됨

## 2. facet 품질 불균일 → 원인 규명 + 측정·개선 (팀 한계 ⑦) ⭐ 가장 강함

- 🔴 **문제**: `facet_json`이 auto_extracted라 **품질이 균일하지 않음** (팀 자백)
- 🔍 **원인 진단**: **본문 빈약 종목(무형·민속·등록)은 evidence가 빈약** → 근거가 얇으면 LLM이 빈틈을 **추측으로 메움** → 충실성 **0.76으로 하락**. *증상(불균일)이 아니라 원인(빈약 본문)을 짚음.*
- ✅ **보완**: RAGAS 충실성으로 약한 facet 탐지 + 프롬프트 강화 → 🔬 환각 회복(초기 **0.76**, 근거 얇을 때 → 강화 후). 빈약군은 보강/제외 처방
- ✅ **측정 완료**(병렬 세션): 이야기·인물까지 3개 관심사 전체 + 정제 버그 교정 최종 함수 반영
  - 🔬 **3개 관심사 전체 충실성 = 0.89** (인물 0.92 / 이야기 0.89 / 건축 0.87, 45건) ⚠️(소표본·Gemini judge) — 건축 위주 15건 0.93보다 표본 3배·더 방어 가능

> ▶ **발표 포인트는 '0.89'가 아니라 "환각을 측정하고 고치는 능력".** 팀은 환각이 무서워 LLM 생성을 *포기*(한계⑤)했지만, 우리는 환각을 **측정**(0.76 탐지)하고 **원인**(빈약 본문)을 짚어 **고쳤다**(→~0.9). → **0.89는 자랑할 헤드라인이 아니라, 그 능력이 실제로 작동한다는 증거(footnote·Q&A).** 숫자 단독으로 들이밀지 말 것.

## 3. 답변이 deterministic이라 뻣뻣함 → grounded LLM 전환 *제안* (팀 한계 ⑤)

- 🔴 **문제**: 로컬 Qwen 환각 때문에 LLM 생성을 포기 → deterministic builder라 **자연스러움 제한**
- 🔍 **원인 진단**: "LLM 생성 = 환각"이 전제였는데, **grounded 프롬프트면 환각 안 함** (원인은 LLM이 아니라 grounding 부재)
- 💡 **제안**(강제 X·논의용): grounded LLM 생성도 🔬 **충실성 ~0.89 유지** → framing만 LLM로 전환하면 자연스러움↑·충실성 유지 가능. 이 수치를 근거로 backend `llm.py` framing 경로 제안

## 4. 데이터에 없는 질문 거절이 모호 → 임계값 실측 + 사각 진단 (팀 목적: 없는/왜곡 질문 차단)

- 🔴 **문제**: 데이터에 없거나 엉뚱한 질문을 **언제 거절할지 기준 불명확**
- 🔍 **원인 진단**: 🔬 임계값 **~0.52**면 해외·딴소리는 거절 가능(정답군 평균 0.647 vs 해외 최대 0.477). 단 **'데이터에 없는 국내 유산'은 유사도가 겹쳐** 임계값만으론 부족 — *사각지대를 정확히 특정함*
- ✅ **한계 넘는 방법**: 1차 임계값(0.52) 거절 → 겹치는 사각지대는 **데이터 보강 + LLM 판별 2차** 로 넘음 (단순 임계값의 한계를 2단으로 돌파)

## 5. 매칭이 텍스트 기반 → 공식 관계키 보완 (팀 한계 ⑥)

- 🔴 **문제**: 유산 매칭이 공식 관계키가 아니라 **텍스트 기반**
- 🔍 **원인 진단**: 메인–부속(경복궁–근정전) **관계가 데이터에 없음** → "근정전이 뭐야?" 같은 부속 질문 약함
- ✅ **보완**: `parent_id` 관계키(4대궁+종묘 수동 매핑)로 부속 직접질문·관계 검색 지원

## 6. 접근성 공백 → 음성 해설 보존 (팀 스키마에 없음)

- 🔴 **문제**: 팀 `heritages` 스키마에 **음성 해설 필드 없음** → 시각약자·어르신 사각
- ✅ **보완**: 📐 `narration_url` 보존 → 음성 해설 제공 (민생 프로젝트 접근성 취지)

## 7. 관심사 4종의 커버리지 한계 → 확장 가능 구조로 설계 (팀 facet 4종)

- 🔴 **문제**: 팀 관심사 **4종**(건축·이야기·인물·답사)은 비석 맥락에서 출발 → 15K 전 종목의 **자연·종교·생활 등 다양성**을 못 담을 수 있음
- 🔍 **원인 진단**: 명승·천연기념물(자연), 무형·민속(생활)은 4종 어디에도 잘 안 잡힘
- ✅ **보완**: 우선 **5종까지 고려**(퀴즈 추가) + nature·religion·life·art 확장 검토. 점수 멀티라벨·재태깅 구조라 **key 추가만으로 확장 가능**(기존 데이터 안 깨짐) 📐
